# Documentation

This folder contains the project notes that support explanation, defense, and final packaging.

Use these files for:

- understanding the overall system design
- explaining safety decisions
- showing how approved medical content is governed
- preparing the final report or presentation

Main documents:

- `architecture.md`: how the frontend, backend, retrieval, and database fit together
- `safety-policy.md`: emergency behavior, refusal rules, and disclaimer policy
- `source-governance.md`: source priority, approval expectations, and content rules
- `limitations.md`: current technical limits, demo constraints, and future upgrade points
